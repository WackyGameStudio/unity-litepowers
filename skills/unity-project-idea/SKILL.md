# Unity Project Idea

Create or update game idea docs through guided questions.

## Process

1. Search `docs/litepowers/project-idea.md`, `game-design.md`, `technical-plan.md`, and `features/**`.
2. Read package doc `docs/output-language-policy.md`.
3. Read package doc `docs/guided-questioning-protocol.md`.
4. Read `references/idea-checklist.md`.
5. Build Known / Assumed / Missing from the user request and docs.
6. Skip questions answered by existing docs or user request.
7. Infer low-risk answers and present recommendation.
8. Ask high-cost or ambiguous decisions one at a time with exactly two context-suitable options, recommendation, reason, and evidence note.
9. Use source/project/web search before asking when option quality depends on facts not in docs.
10. Do not invent filler choices; if only one option is viable, ask for confirmation.
11. Save to:
   - `docs/litepowers/project-idea.md` when creating first main docs.
   - `docs/litepowers/features/YYYY-MM-DD-feature-name/iterations/YYYY-MM-DD-HHmmss-idea.md` for feature or repeated planning.
12. Use `feature.md` to confirm same feature identity; create `-02`, `-03` suffix folder when identity differs or is unclear.
13. Update canonical `idea.md` only after approval or explicit update condition.
14. Mark main docs update needs.
15. End with a Handoff block that names `unity-game-planning` when ready.
