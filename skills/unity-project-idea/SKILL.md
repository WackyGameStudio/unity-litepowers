# Unity Project Idea

Create or update game idea docs through guided questions.

## Process

1. Search `docs/litepowers/project-idea.md`, `game-design.md`, `technical-plan.md`, and `features/**`.
2. Read `references/idea-checklist.md`.
3. Skip questions answered by existing docs or user request.
4. Infer low-risk answers and present recommendation.
5. Ask high-cost or ambiguous decisions one at a time.
6. Save to:
   - `docs/litepowers/project-idea.md` when creating first main docs.
   - `docs/litepowers/features/YYYY-MM-DD-feature-name/iterations/YYYY-MM-DD-HHmmss-idea.md` for feature or repeated planning.
7. Use `feature.md` to confirm same feature identity; create `-02`, `-03` suffix folder when identity differs or is unclear.
8. Update canonical `idea.md` only after approval or explicit update condition.
9. Mark main docs update needs.
10. End with a Handoff block that names `unity-game-planning` when ready.
